# Security Policy

## Supported Versions

The following table outlines which versions of **Starlight Programming Language** are currently supported with security updates:

| Version | Supported |
|--------|-----------|
| 0.x    | ✅ Yes    |
| < 0.x  | ❌ No     |

> **Note:** Starlight is currently in early development. APIs and internals may change rapidly.

---

## Reporting a Vulnerability

If you discover a security vulnerability in Starlight, please report it responsibly.

### How to Report
- **Do not** open a public GitHub issue for security vulnerabilities.
- Send details via **GitHub Security Advisories** (preferred), or
- Contact the maintainer directly via GitHub.

### What to Include
Please include as much information as possible:
- A clear description of the vulnerability
- Steps to reproduce
- A minimal proof-of-concept (if available)
- Potential impact and severity
- Suggested fixes (optional)

---

## Scope

This security policy applies to:
- Starlight language runtime
- CLI tool (`starlight`)
- Official documentation and examples

Third-party dependencies and external packages should be reported to their respective maintainers.

---

## Disclosure Process

1. Vulnerability is reported privately
2. Issue is verified and assessed
3. Fix is developed and tested
4. Security patch is released
5. Public disclosure (if applicable)

---

## Security Best Practices

While using Starlight:
- Avoid executing untrusted scripts
- Review imported modules carefully
- Keep dependencies up to date
- Run scripts with least-privilege permissions
